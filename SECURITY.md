# Security Policy

## Purpose

ZeroDaySentinel is a defensive cybersecurity research project. It is designed for authorized education, detection engineering, threat modeling, incident response planning, and responsible disclosure workflows.

This repository does not provide real-world exploit code, weaponized payloads, bypass techniques, persistence mechanisms, stealth logic, or instructions for unauthorized access.

## Responsible Use

This project may be used only for:

- authorized lab environments,
- defensive security research,
- blue-team training,
- security awareness,
- incident response preparation,
- responsible disclosure documentation.

This project must not be used for:

- unauthorized testing,
- exploitation of real systems,
- internet-wide scanning,
- payload development,
- persistence or evasion research,
- credential theft,
- disruption, damage, or unauthorized access.

## Reporting Security Issues

If you find a security issue related to this repository, please report it privately through GitHub security reporting or by contacting the maintainer.

Do not publicly disclose sensitive details before the issue has been reviewed.

## Safe Research Principles

ZeroDaySentinel follows these principles:

1. Defensive analysis only.
2. No real exploit release.
3. No weaponized proof-of-concept code.
4. No unauthorized testing.
5. No stealth, persistence, or evasion logic.
6. Synthetic examples only.
7. Responsible disclosure encouraged.
