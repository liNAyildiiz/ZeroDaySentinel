# Contributing to ZeroDaySentinel

Thank you for your interest in contributing to ZeroDaySentinel.

This project welcomes contributions related to defensive cybersecurity, detection engineering, incident response, threat modeling, responsible disclosure, documentation, and safe simulation.

## Allowed Contributions

Good contributions include:

- improving documentation,
- adding defensive detection examples,
- improving synthetic log generation,
- adding Sigma-style or YARA-style defensive rules,
- improving incident response playbooks,
- improving responsible disclosure templates,
- adding safe test cases,
- improving code quality.

## Not Allowed

The following contributions will not be accepted:

- real-world exploit code,
- weaponized payloads,
- unauthorized testing instructions,
- exploit chaining,
- stealth or persistence mechanisms,
- detection evasion techniques,
- credential theft logic,
- internet-wide scanning tools,
- content that enables harm against real systems.

## Development Principles

Before contributing, please make sure your change is:

- defensive,
- ethical,
- non-weaponized,
- reproducible,
- clearly documented,
- safe to run locally.

## Pull Request Checklist

Before opening a pull request, check that:

- the contribution does not contain exploit code,
- the contribution does not target real systems,
- documentation is clear,
- examples are synthetic,
- the change supports defensive learning or readiness.
